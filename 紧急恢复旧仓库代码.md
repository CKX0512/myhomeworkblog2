# 🚨 紧急恢复旧仓库代码指南

## 情况说明
刚才的推送覆盖了 `CKX0512/my-homework-blog` 仓库中的旧代码。以下是恢复方法：

---

## 方法 1: 在 GitHub 上直接恢复（最快）

### 步骤：

1. **访问你的仓库**
   - 打开：https://github.com/CKX0512/my-homework-blog
   - 点击 **"Commits"** 或查看提交历史

2. **找到被覆盖之前的最后一个提交**
   - 在提交历史中找到你原来的代码对应的提交
   - 点击那个提交的 SHA 值（例如：`abc1234`）

3. **恢复代码**
   - 在提交详情页面，点击右上角的 **"Browse files"** 按钮
   - 或者直接访问：`https://github.com/CKX0512/my-homework-blog/tree/提交SHA`
   - 点击 **"Code"** 按钮 → **"Download ZIP"** 下载旧代码

4. **强制推送恢复**
   ```bash
   # 在本地创建一个新分支来恢复
   git checkout -b restore-old-code
   
   # 删除当前所有文件（除了.git）
   # 然后解压下载的ZIP，复制所有文件过来
   
   # 提交恢复
   git add .
   git commit -m "恢复旧代码"
   
   # 强制推送到main分支（⚠️ 会覆盖当前代码）
   git checkout main
   git reset --hard restore-old-code
   git push origin main --force
   ```

---

## 方法 2: 使用 Git Revert（如果知道具体提交）

如果你知道是哪个提交覆盖了旧代码：

```bash
# 1. 重新添加远程仓库
git remote add origin https://github.com/CKX0512/my-homework-blog.git

# 2. 拉取最新代码
git fetch origin

# 3. 查看提交历史，找到要回退的提交
git log --oneline

# 4. 回退到指定提交（替换 COMMIT_HASH 为旧代码的提交哈希）
git reset --hard COMMIT_HASH

# 5. 强制推送（⚠️ 会覆盖远程代码）
git push origin main --force
```

---

## 方法 3: 从 GitHub 克隆旧版本

1. **在 GitHub 上找到旧提交**
   - 访问仓库的 Commits 页面
   - 找到旧代码的提交

2. **克隆特定提交**
   ```bash
   # 创建一个临时目录
   cd ..
   mkdir temp-restore
   cd temp-restore
   
   # 克隆仓库
   git clone https://github.com/CKX0512/my-homework-blog.git
   cd my-homework-blog
   
   # 切换到旧提交（替换 OLD_COMMIT_HASH）
   git checkout OLD_COMMIT_HASH
   
   # 复制所有文件回原项目目录
   # （手动复制或使用命令）
   ```

---

## 方法 4: 使用 GitHub 的恢复功能

1. **在 GitHub 仓库页面**
   - 点击 **"Insights"** → **"Network"** 查看分支图
   - 或者点击 **"Code"** → 选择不同的分支/标签

2. **创建新分支保护旧代码**
   - 在旧提交处创建新分支
   - 然后合并回 main 分支

---

## ⚠️ 重要提醒

1. **强制推送 (`--force`) 会覆盖远程代码，请谨慎使用**
2. **建议先创建一个备份分支**：
   ```bash
   git branch backup-before-restore
   ```
3. **如果仓库有协作者，强制推送前先通知他们**

---

## 如果以上方法都不行

如果你有：
- 本地其他位置的备份
- 其他 Git 仓库的备份
- 代码压缩包

可以直接用那些文件覆盖当前代码，然后提交。
